# dotty v0.3.0

Draft release notes for the first public-ready dotty release.

## Highlights

- Layered dotfiles chains via `DOTTY_EXTENDS`, with later repos overriding earlier repos.
- Environment overlays through `$ENV/home/` when a repo needs laptop or remote-specific files.
- Debugging and maintenance commands including `dotty status`, `dotty files`, `dotty trace`, and `dotty doctor`.
- Repo-defined `dotty run` commands and one-shot cleanup tasks under `.dotty/cleanups/`.
- Guard tooling, operation locking for mutating commands, and opt-in parallel pulls during `dotty update`.
- Public examples and contributor docs for personal-plus-work repo setups.

## Install notes

- The default curl installer intentionally continues to bootstrap from `main` for this first public-ready release:

  ```bash
  curl -fsSL https://raw.githubusercontent.com/jackokerman/dotty/main/install.sh | bash
  ```

- This keeps first-time install simple while the release process stays lightweight.
- Users who want a pinned checkout can clone the repo and check out `v0.3.0` before running dotty from that checkout.
